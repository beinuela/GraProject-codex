# 答辩项目演示流程卡片

适合答辩当天放在旁边快速查看。完整版本见 `docs/答辩项目演示完整流程.md`。

## 演示前 3 分钟检查

- MySQL 已启动，库名：`campus_material`
- 后端：`cd backend && mvn spring-boot:run -Dspring-boot.run.profiles=prod`
- 前端：`cd frontend && npm run dev`
- 健康检查：`http://127.0.0.1:18080/actuator/health` 返回 `UP`
- 浏览器打开：`http://localhost:5173/login`
- 页面显示“系统首页”“数据看板”等正式文案
- 准备备用截图或录屏

## 账号

| 账号 | 角色 | 用途 |
| --- | --- | --- |
| `admin` | 系统管理员 | 总览、基础数据、日志、统计 |
| `dept` | 部门用户 | 新建申领、提交、签收 |
| `approver` | 审批人员 | 审批申领、审批调拨 |
| `warehouse` | 仓库管理员 | 入库、出库、调拨执行、预警处理 |

演示密码使用初始化脚本中的演示密码。正式部署必须替换。

## 7 分钟推荐流程

| 时间 | 页面 | 讲什么 |
| --- | --- | --- |
| 0:00 - 0:40 | 登录页 | JWT 登录、角色菜单、权限边界 |
| 0:40 - 1:20 | 系统首页 | 指标、库存图表、未处理预警 |
| 1:20 - 2:00 | 用户/角色/物资/仓库 | 基础数据支撑业务流程 |
| 2:00 - 2:50 | 库存查询/入库 | 汇总库存、批次、有效期、入库同步 |
| 2:50 - 4:30 | 申领审批/出库 | 申请、提交、锁定库存、审批、出库、签收 |
| 4:30 - 5:20 | 调拨管理 | 跨仓调拨、推荐调出仓、执行和签收 |
| 5:20 - 6:20 | 预警管理/通知日志 | 低库存、临期、过期、异常出库、审计 |
| 6:20 - 7:00 | 统计分析 | ECharts、趋势分析、补货建议、总结 |

## 必讲业务闭环

部门用户提交申领 -> 系统检查并锁定库存 -> 审批人员通过 -> 仓库管理员出库 -> 部门用户签收 -> 首页、日志、统计数据更新 -> 库存不足或临期时进入预警处理。

## 稳妥话术

“本系统的智能主要体现在规则预警、需求趋势分析、补货建议和数据可视化辅助决策，不是大型机器学习平台。”

“当前系统没有独立配送任务模块，演示中把物资交付体现在出库、调拨执行和签收状态中，后续可以扩展为独立配送流程。”

“如果现场数据较少，图表会比较平缓；正式使用后，随着申领、入库、出库和调拨记录增加，统计分析会更有参考价值。”

## 常见故障快修

```powershell
netstat -ano | findstr :8080
taskkill /PID <PID> /F
```

```powershell
netstat -ano | findstr :5173
taskkill /PID <PID> /F
```

登录失败：确认已导入 `sql/seed.sql`，优先使用 `admin`。

页面空数据：清空筛选条件，回到已有演示数据。

预警扫描提示过快：不要连续点击，这是接口限流保护。
