# 项目正式环境功能测试与答辩演示流程报告

## 1. 测试说明

本次测试目标是使用浏览器对“校园物资智能管理系统”进行正式环境功能测试，并整理可用于毕业设计答辩的项目演示流程。测试方式包括：正式环境配置核验、生产构建前端页面访问、浏览器逐菜单截图、多角色登录验证、非破坏性新增演示数据、核心业务流程验证、问题复现和截图记录。

测试时间：2026-05-10  
测试地点：本机正式环境  
前端访问地址：`http://127.0.0.1:4173`  
后端接口地址：`http://127.0.0.1:8080`  
管理端口：`http://127.0.0.1:18080`  
测试数据策略：使用 `正式环境演示-*`、`demo_*` 等新增演示数据，不删除真实数据，不修改原始用户、原始库存和原始审批记录。

说明：本机未安装 Nginx，正式部署文档要求的 Nginx 静态资源服务和 `/api` 反向代理由本次本地静态代理临时代替；前端资源为 `npm run build` 生成的生产构建，不使用 Vite dev server。后端以 `prod` profile 启动，连接 MySQL 正式库。

## 6. 核心业务闭环测试

### 6.1 用户要求闭环

用户要求闭环为：

提交需求 → 审批 → 分配库存 → 生成配送任务 → 派单 → 签收 → 库存变化 → 通知日志 → 统计分析

当前正式环境无法完整跑通该闭环，原因如下：

| 节点 | 测试结果 | 说明 |
|---|---|---|
| 普通用户提交物资申领 | 不通过 | 申领接口 500，正式库缺少 `reserved_warehouse_id` 字段 |
| 审批人员审批申领 | 未能继续 | 前置申领单创建失败 |
| 仓库管理员分配库存/出库 | 未能按申领继续 | 申领关联出库依赖成功申领单 |
| 生成配送任务 | 未实现 | 系统无配送/派单模块 |
| 调度人员派单 | 未实现 | 系统无调度员角色和派单页面 |
| 配送/签收 | 未实现独立配送 | 只有申领签收和调拨签收概念 |
| 库存变化 | 部分验证 | 入库成功，调拨执行成功；独立出库失败 |
| 通知/日志 | 通过 | 事件通知、操作日志、登录日志可查 |
| 统计分析更新 | 通过 | 统计页面可刷新并展示图表 |

## 9. 测试结论

当前项目具备部分答辩演示条件，但不具备“完整申领配送闭环”的稳定演示条件。

可以稳定展示的内容：

- 登录页、错误密码提示、未登录拦截；
- 首页 KPI、图表和指挥大屏；
- 用户、角色、物资、仓库等基础资料；
- 入库和批次查看；
- 调拨智能推荐、审批、执行、签收闭环；
- 预警扫描与处理；
- 事件上报、通知中心、操作日志、登录日志；
- 统计分析图表。

需要规避或修复后再演示的内容：

- 申领审批闭环：正式库缺字段导致接口 500；
- 配送/派单：未实现；
- 独立出库：出现批次库存不足；
- 退出登录：跳转不稳定；
- 非管理员直达敏感路由：页面级权限体验需优化。

客观结论：建议答辩现场以“库存基础数据 + 入库批次 + 预警处理 + 调拨闭环 + 统计分析 + 权限菜单差异”为主线演示；不要展示申领配送闭环，除非先完成正式库迁移和相关功能修复。
