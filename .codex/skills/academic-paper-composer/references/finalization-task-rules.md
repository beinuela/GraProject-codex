# Final Thesis Delivery Rules

Use this reference when turning an existing draft into a submission-ready `.docx`.

## Mandatory Sequence

1. Copy the original draft.
2. Read the school format sample completely.
3. Rework the manuscript using the real project evidence.
4. Redraw or correct required figures.
5. Apply final Word formatting on the copied file.
6. Produce a separate rework report.

## File Safety

- Never overwrite the original draft.
- Only edit the copied destination file.

## Final Manuscript Must Not Contain

- revision comments
- review explanations
- process notes
- `【需补证据】` style placeholders
- `【建议补图】` style placeholders
- template prompts
- text box instructions
- messages intended only for the author

## Visual Constraints

- Headings and body text must be black.
- No blue titles.
- Remove hyperlink styling from visible text.
- Figures must be black/white or grayscale only.
- Figures must use software engineering thesis style:
  - white background
  - black/gray lines
  - no infographic style
  - no decorative elements

## Allowed Figure Types

- system architecture
- functional module
- business process
- sequence
- E-R
- use case

## Evidence Constraints

- Base the thesis on the actual project only.
- Do not invent functions, interfaces, tables, or metrics.
- Lower "innovation" to engineering highlights.
- Keep the testing chapter rigorous and conservative.

## Separate Rework Report

The report may include:
- deleted or weakened content
- unsupported data not retained
- manual verification points
- figure rebuild notes
- formatting fixes

None of that may appear in the final thesis body.

## Final Checklist

- copied before editing
- original draft preserved
- all text black
- no blue headings
- all figures grayscale
- all figures engineering-style
- no process text remains
- school sample followed
- automatic TOC present
- ready for submission
