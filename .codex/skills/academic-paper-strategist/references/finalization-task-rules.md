# Final Thesis Delivery Rules

Use this reference when the task is to turn an existing draft into a submission-ready undergraduate thesis DOCX.

## Mandatory Execution Order

1. Copy the draft first.
2. Read the school format sample completely.
3. Rework the thesis body from draft + project evidence.
4. Redraw or correct thesis figures.
5. Apply final DOCX formatting on the copied file.
6. Write a separate rework report.

Do not reorder these steps without a strong, explicit reason.

## File Safety

- Never overwrite the original draft.
- Always edit a copied file at the user-specified destination.
- If the destination already exists, confirm whether it should be replaced only when required by the broader task constraints. Otherwise create/update only the copy path the user requested.

## Final Manuscript Prohibitions

The final thesis body must not contain:
- revision notes
- review comments
- process markers
- "needs evidence" placeholders
- "suggested figure/table" placeholders
- template instructions
- instructional text boxes
- explanations intended for the author rather than the examiner

## Visual Rules

- All visible headings and body text must be black.
- Remove hyperlink styling from headings and normal text.
- Figures must be engineering-paper style:
  - white background
  - black or gray lines
  - no color-coded infographic style
  - no decorative elements
- Only these figure types are allowed:
  - system architecture
  - functional module
  - business process
  - sequence
  - E-R
  - use case

## Evidence Rules

- Base the final text on the actual project implementation.
- Do not invent interfaces, database objects, metrics, or deployment claims.
- Downgrade "innovation" to engineering highlights.
- Keep the testing chapter conservative and evidence-based.

## Required Outputs

Produce:
- final DOCX
- separate rework report

The rework report may contain:
- what was removed or weakened
- what lacked evidence
- what still needs manual review
- figure rebuild notes
- formatting correction notes

The final DOCX may not contain any of those process notes.

## Final Self-Check

Before delivery, check all of the following:
- source draft was copied before editing
- original draft was not overwritten
- all headings are black
- there are no blue headings
- all figures are black/white or grayscale
- figures match engineering thesis style
- no process text remains in the manuscript
- school format sample was followed
- TOC is automatic
- the final document is submission-ready
